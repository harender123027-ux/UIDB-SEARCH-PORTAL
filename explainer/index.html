Pehchan (Unidentified Bodies Identification System) — User flow, architecture & tech (Phase 1 MVP)

Run the app
Backend: cd backend && .venv/bin/uvicorn app.main:app --reload --port 8000
Frontend: npm run dev → http://localhost:5173
Seed repository: add images to backend/reference_photos/ then python -m scripts.seed_demo_repository

User flow (high level)

1. Dashboard — view totals, recent cases 2. New Case — register unidentified body (photos + attributes) → creates submission 3. Search — add photo and/or description and/or voice → click Search → see matches from repository 4. Schemas — reference view of data models 5. Audit Log — immutable trail of actions

Architecture

Frontend (PWA)

React SPA (Vite). Single-page app with tabs: Dashboard, New Case, Search, Schemas, Audit. Calls backend REST API. No auth in MVP.

Backend (API)

FastAPI. Handles submissions, face embedding, vector search, attribute/voice search. Persists to SQLite and Qdrant.

Data flow

Photo upload → InsightFace embedding (512-d) → Qdrant vector DB Text/voice → keyword extraction → attribute filter on reference_persons Combined search → merge by overlap (how many inputs matched) → sort by confidence

Repository

Reference persons (missing persons) stored in SQLite + face embeddings in Qdrant. Seed script loads images from reference_photos/ and indexes them.

Dashboard

Shows total cases, cases with matches, and recent submissions table. Empty state when no data.

New Case

Multi-step: (1) Photos — face frontal, profiles, full body, marks; face condition. (2) Physical attributes. (3) Submit. Backend extracts face embeddings and stores in Qdrant.

Search

One simple flow: Add any combination of photo, text description, or voice note. Click Search. All inputs are run against the repository; results merged by overlap (how many modalities matched) and confidence. Voice is transcribed via Whisper before search.

Schemas

Reference view: PostgreSQL tables (cases, persons, images, matches, feedback, missing_persons, users, audit_log), Qdrant collections, Blob storage layout.

Audit Log

INSERT-only trail. Every submission, match, feedback recorded with timestamp, action, resource.
